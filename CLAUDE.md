## Project Overview

"Star Catcher" — a single-file HTML5 canvas game wrapped with Capacitor for iOS deployment. The visual style is 80's monochrome CRT (black background, white wireframe graphics, scanline overlay).

## Architecture

- **`index.html`** — The entire game: CSS, HTML, and JS in one file. Contains the game loop, canvas rendering, input handling, particle system, and UI overlay. This is the source of truth.
- **`www/`** — Capacitor's web asset directory. Must contain a copy of `index.html` before syncing to iOS. This is NOT the source file — always edit `index.html` at project root, then sync.
- **`ios/`** — Native Xcode project generated by Capacitor. Do not hand-edit; managed via `npx cap` commands.
- **`capacitor.config.json`** — Capacitor config (app ID: `com.starcatcher.app`, webDir: `www`).

## Common Commands

```bash
# After editing index.html, copy to www/ and sync to iOS
cp index.html www/ && npx cap copy ios

# Full sync (also updates plugins/native dependencies)
cp index.html www/ && npx cap sync ios

# Open Xcode project (build/run from there)
npx cap open ios
```

## Key Conventions

- **Monochrome palette only**: all canvas drawing uses `#fff`, `#ccc`, `#ddd` on black. No colors.
- **Wireframe style**: shapes are stroked, not filled (stars, character). The character is a stick figure.
- **CRT effect**: CSS scanlines via `#gameContainer::after` pseudo-element.
- **Font**: `"Courier New", monospace` in both CSS and canvas text.
- No build tooling or bundler — the game is vanilla HTML/CSS/JS.
