## Project Overview

"Star Catcher" — a single-file HTML5 canvas game wrapped with Capacitor for iOS deployment. The visual style is 80s monochrome CRT (black background, white pixel art, scanline overlay).

## Architecture

- **`index.html`** — The entire game: CSS, HTML, and JS in one file. Contains the game loop, canvas rendering, input handling, particle system, and UI overlay. This is the source of truth.
- **`www/`** — Capacitor's web asset directory. Must contain a copy of `index.html` before syncing to iOS. This is NOT the source file — always edit `index.html` at project root, then sync.
- **`ios/`** — Native Xcode project generated by Capacitor. Do not hand-edit; managed via `npx cap` commands.
- **`capacitor.config.json`** — Capacitor config (app ID: `com.starcatcher.app`, webDir: `www`).

## Rendering Pipeline

The game uses a two-canvas approach for crisp pixel art:
1. **Offscreen canvas** (`W × H`, ~120×175) — all game elements are drawn here at low resolution.
2. **Display canvas** (`W*SCALE × H*SCALE`, ~480×700) — the offscreen canvas is copied here with `imageSmoothingEnabled = false` for nearest-neighbor upscaling.
3. **CSS scaling** — the display canvas uses `width: 100vw; height: 100vh; object-fit: contain; image-rendering: pixelated` to fill the viewport while maintaining crisp pixels.

`SCALE = 4` means each game pixel is 4×4 canvas pixels. CSS handles further scaling to the viewport.

## Sprites

Sprites are defined as 2D arrays of 0s and 1s. Each 1 is drawn as a single pixel on the offscreen canvas.
- `PLAYER_SPRITE` — 11×12 human figure
- `STAR_SPRITES` — two 5×5 frames for animation
- `LIFE_SPRITE` — 3×3 star for HUD lives display

## Common Commands

```bash
# After editing index.html, copy to www/ and sync to iOS
cp index.html www/ && npx cap copy ios

# Full sync (also updates plugins/native dependencies)
cp index.html www/ && npx cap sync ios

# Open Xcode project (build/run from there)
npx cap open ios
```

## Key Conventions

- **Monochrome palette only**: all canvas drawing uses `#fff`, `#ccc`, `#888` on black. No colors.
- **Pixel art style**: sprites are 2D arrays drawn pixel-by-pixel. Shapes are filled, not stroked.
- **CRT effect**: CSS scanlines via `body::after` pseudo-element.
- **Font**: `"Press Start 2P"` for UI overlay, `monospace` for canvas HUD text.
- No build tooling or bundler — the game is vanilla HTML/CSS/JS.
- `www/` and `ios/` are gitignored — they are derived from `index.html`.
